# Demo Dataset Guide

This demo pack is built for the supervisor productivity workflow, not analytics-first exploration.

Files generated by [scripts/generate_demo_dataset.py](/Users/joeschloemer/Desktop/dpd_web/scripts/generate_demo_dataset.py):

- `demo_data/demo_supervisor_history.csv`
- `demo_data/demo_actions_seed.json`
- `demo_data/demo_action_events_seed.json`
- `demo_data/demo_storytelling.json`

## Dataset Shape

- Employees: 32
- History: 18 days
- Pre-seeded actions: 9
- Required scenarios covered:
  - 2 overdue follow-ups
  - 2 due today
  - 2 repeat no-improvement cases
  - 2 recognition opportunities
  - 1 resolved win

## CSV Structure

Import CSV columns:

| Column | Purpose |
| --- | --- |
| `Date` | Work date in `YYYY-MM-DD` |
| `Department` | Picking, Packing, or Receiving |
| `EmployeeID` | Stable demo employee identifier |
| `EmployeeName` | Friendly display name |
| `Shift` | `Day` or `Night` |
| `UPH` | Units per hour |
| `Units` | Rounded unit count |
| `HoursWorked` | Shift hours |

## Action Event Seed Data

`demo_action_events_seed.json` contains the immutable history for those actions.

Event patterns included:

- `created` only for recognition opportunities so they still appear as open recognition work
- `created -> coached -> follow_up_logged` for due-today and overdue follow-ups
- repeated `coached` plus `no_change`/`worse` outcomes for repeat no-improvement cases
- `created -> coached -> follow_up_logged -> resolved` for the resolved win

1. Jamie Carter: “This is an overdue follow-up. The supervisor already coached Jamie, but the loop was never closed. Today surfaces that immediately.”
